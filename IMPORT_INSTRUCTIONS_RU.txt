ИНСТРУКЦИЯ ДЛЯ АГЕНТА AI STUDIO

Это восстановленный проект. Файлы package.json, app/, components/, lib/ и types/ должны находиться прямо в корне рабочей директории.

1. Сначала выполни только диагностику:
   pwd
   ls -la
   test -f package.json && echo PACKAGE_OK
   test -f app/page.tsx && echo APP_OK
   test -d components && echo COMPONENTS_OK

2. Если PACKAGE_OK или APP_OK не выводятся, НЕ ЧИНИ NEXT.JS и НЕ СОЗДАВАЙ НОВЫЕ ФАЙЛЫ-ФИКСЫ. Сначала перемести содержимое распакованного архива в корень текущего workspace. В корне должны одновременно находиться package.json и папка app.

3. Не создавай файлы fix-*.js, patch-*.js, update-*.js, replace*.js, временные конфиги и копии package.json.

4. После подтверждения, что исходники находятся в корне:
   rm -rf node_modules .next
   npm ci
   npm run lint
   npm run dev

5. Проверь GET / и убедись, что возвращается HTTP 200.

6. Версии зависимостей уже зафиксированы. Не обновляй Next.js до 15.5.x или Next.js 16. Не добавляй символы ^ или ~.

7. Сохрани все существующие API-маршруты и компоненты. Не переписывай приложение и не удаляй функциональность.

Путь /app/applet в логах является внутренней рабочей директорией контейнера и сам по себе не является ошибкой. Ошибка была в том, что исходники не были помещены в корень этой директории.
